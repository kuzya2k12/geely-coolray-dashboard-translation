# Установка и запуск

В репозитории — только наш код и документация. Дампы прошивки, SDK Infineon и его
энкодер **не включены** (авторское право). Ниже — как их получить и как запустить.

## 1. Зависимости

```
pip install -r requirements.txt      # capstone + pillow
```
Python 3.9+.

## Проверка декодера (без дампов и без энкодера)

Корректность RLAD-декодера можно проверить сразу — тест-векторы зашиты в репозиторий:
```
python tests/test_rlad_decoder.py        # или: python -m pytest tests/
```
Должно быть `8/8 passed`. Это доказывает, что декодер работает, без проприетарного
`rlad-encoder.exe`.

## 2. Что нужно достать отдельно

| Что | Зачем | Где взять |
|-----|-------|-----------|
| Дампы NOR приборки (`*.bin`, 2×64 МБ) | входные данные | готовые (китайский интерфейс): **[Яндекс.Диск](https://disk.yandex.by/d/vSQ8tQrcU28Sog)** — или снять со своей приборки (программатор Dediprog/Elnec, чип S25HL512T, BGA-24, 3.0 В) |
| Референс-энкодер RLAD (`rlad-encoder.exe`) | «оракул» для проверки/пересборки RLAD | [github.com/Infineon/mtb-example-psoc-edge-gfx-rlad](https://github.com/Infineon/mtb-example-psoc-edge-gfx-rlad), файл `utility/rlad-encoder.exe` |
| **ResourceGenerator.exe** + Bin2Text.exe | генератор ресурсов Infineon (оракул для формата surface/контейнера) | [github.com/Infineon/mtb-t2g-example-graphics-sample-drawing](https://github.com/Infineon/mtb-t2g-example-graphics-sample-drawing), путь `tool/graphics/bin/windows/` |
| Заголовки Infineon (`*.h`) | справка по структурам форматов | [github.com/Infineon/tviic2d-gfx-mw](https://github.com/Infineon/tviic2d-gfx-mw) (учтите EULA Infineon) |
| Библиотеки Infineon (`*.a`) | для дизассемблера (восстановление формата) | тот же репозиторий `tviic2d-gfx-mw` |

> **Открытая экосистема Infineon TRAVEO — основа этого проекта.** Именно потому, что
> SDK и инструменты (`rlad-encoder`, `ResourceGenerator`, `tviic2d-gfx-mw`) выложены
> публично, удалось восстановить формат RLAD. Полный разбор релевантных репозиториев
> Infineon (что в них есть и чего принципиально нет) — в
> [INFINEON_REPOS.md](INFINEON_REPOS.md).

Разложите так (пути, которые ожидает код):
```
<рабочая папка>/
  Coolray_25_Dash_U17.bin        # ваши дампы
  Coolray_25_Dash_U18.bin
  rlad-ex/utility/rlad-encoder.exe
  infineon-gfx/                  # клон tviic2d-gfx-mw
  geely-coolray-dashboard-translation/   # этот репозиторий
```

> Пути в скриптах прописаны относительными (`../Coolray_...`, `../rlad-ex/...`).
> При другом расположении поправьте пути в начале соответствующего скрипта.

## 3. Быстрая проверка декодера (без своей машины)

Декодер RLAD можно проверить round-trip’ом через энкодер-оракул:
```
python tools/oracle.py        # кодирует тест-картинки энкодером, показывает байты
```
Модуль декодера: `geely_cluster/rlad_decoder.py` — функция `decode_image`.

## 4. Разбор дампа (исследовательские скрипты)

- `tools/analyze.py <dump.bin>` — карта энтропии, сигнатуры, строки.
- `tools/disasm_rlc.py` — дизассемблер функций Infineon (нужны `*.a`).
- `tools/scan_rld.py`, `tools/hunt_glyphs.py`, `tools/find_surfaces.py` — детекторы
  регионов (исследовательские; см. INVESTIGATION_HISTORY — они дают шум без таблицы
  ресурсов).
- `tools/render_region.py <dump.bin> <offset_hex> <size_kb>` — рендер региона в PNG.

### Инструменты сессии 2026-07 (визуальный поиск НЕсжатого контента)

Ключевой вывод сессии: **несжатые ресурсы (raw A8/A1) читаются напрямую при верной ширине
(stride)**; так извлечены иконки приборки (raw A8, stride 48). RLAD-сжатое вслепую не берётся.

- **`tools/extract_raw_a8.py <dump.bin>`** — извлекает несжатые raw-A8 иконки: детектор зоны
  (по числу уникальных байт), автокорреляция ширины, нарезка на отдельные иконки + панорама.
  Даёт подтверждённую библиотеку иконок (зона `0x28F8000`, stride 48). См. SUCCESSFUL_FINDINGS.md.
- **`tools/dump_to_images.py`** — ГЛАВНЫЙ инструмент ручного просмотра. Весь дамп → большие 16:9
  полотна во всех форматах (A8/A1/RGB565/RGB888/RGBA) и на всех ширинах (одна папка на режим,
  ширина+адрес в имени файла). Цветные монохромные полотна пропускаются. Инструкция запуска —
  в `BRUTE_SEARCH_README.md` и заголовке скрипта. Критично: **ширина = stride ресурса**, без
  верной ширины растр «съезжает» (иконки читаются ТОЛЬКО на 48; текст мог бы быть на своей).
- **`tools/find_glyph_grid.py`** — автокорреляционный детектор регулярной сетки глифов (шрифт:
  латиница/цифры, CJK 16/24/32) по несжатым зонам. Выдаёт отчёт с адресами-кандидатами.
  Оговорка: на RLAD-зонах даёт ложные срабатывания (регулярность упаковки ≠ глифы) — проверять
  distinct-bytes (<64 = сыро) и визуально.
- **`tools/brute_raw_glyphs.py`** — автономный многочасовой брутфорс несжатых глифов (форматы ×
  ширины × стартовые смещения) со скорингом «глифо-подобности» и ранжированной галереей.
  Только для несжатых зон (RLAD не берёт by design).

> Все эти инструменты бьют по НЕсжатым зонам. Для RLAD-сжатого (текст, цвет, крупная графика)
> нужен старт+ширина из таблицы разметки MCU (JTAG XIP) или diff двух языков — см. ниже §5 и
> INVESTIGATION_HISTORY.md §3a.

## 5. Рекомендуемый путь (diff двух языков)

Если у вас есть дамп тех же чипов на другом языке — сравните побайтово с китайским:
```
cmp -l chinese_U17.bin translated_U17.bin   # или python-скрипт diff
```
Изменённые регионы = строки/глифы. К ним применяйте `rlad_decoder`. Это самый
надёжный путь для платформы A5 (вся приборка в 128 МБ, вся разница — в этих чипах).
