# Установка и запуск

В репозитории — только наш код и документация. Дампы прошивки, SDK Infineon и его
энкодер **не включены** (авторское право). Ниже — как их получить и как запустить.

## 1. Зависимости

```
pip install capstone pillow
```
Python 3.9+.

## 2. Что нужно достать отдельно

| Что | Зачем | Где взять |
|-----|-------|-----------|
| Дампы NOR приборки (`*.bin`, 2×64 МБ) | входные данные | снять со своей приборки (программатор Dediprog/Elnec, чип S25HL512T, BGA-24, 3.0 В) |
| Референс-энкодер RLAD (`rlad-encoder.exe`) | «оракул» для проверки/пересборки | открытый пример Infineon: `github.com/Infineon/mtb-example-psoc-edge-gfx-rlad`, файл `utility/rlad-encoder.exe` |
| Заголовки Infineon (`*.h`) | справка по структурам форматов | `github.com/Infineon/tviic2d-gfx-mw` (учтите EULA Infineon) |
| Библиотеки Infineon (`*.a`) | для дизассемблера (восстановление формата) | тот же репозиторий Infineon |

Разложите так (пути, которые ожидает код):
```
<рабочая папка>/
  Coolray_25_Dash_U17.bin        # ваши дампы
  Coolray_25_Dash_U18.bin
  rlad-ex/utility/rlad-encoder.exe
  infineon-gfx/                  # клон tviic2d-gfx-mw
  geely-coolray-dashboard-translation/   # этот репозиторий
```

> Пути в скриптах прописаны относительными (`../Coolray_...`, `../rlad-ex/...`).
> При другом расположении поправьте пути в начале соответствующего скрипта.

## 3. Быстрая проверка декодера (без своей машины)

Декодер RLAD можно проверить round-trip’ом через энкодер-оракул:
```
python tools/oracle.py        # кодирует тест-картинки энкодером, показывает байты
```
Модуль декодера: `geely_cluster/rlad_decoder.py` — функция `decode_image`.

## 4. Разбор дампа (исследовательские скрипты)

- `tools/analyze.py <dump.bin>` — карта энтропии, сигнатуры, строки.
- `tools/disasm_rlc.py` — дизассемблер функций Infineon (нужны `*.a`).
- `tools/scan_rld.py`, `tools/hunt_glyphs.py`, `tools/find_surfaces.py` — детекторы
  регионов (исследовательские; см. INVESTIGATION_HISTORY — они дают шум без таблицы
  ресурсов).
- `tools/render_region.py <dump.bin> <offset_hex> <size_kb>` — рендер региона в PNG.

## 5. Рекомендуемый путь (diff двух языков)

Если у вас есть дамп тех же чипов на другом языке — сравните побайтово с китайским:
```
cmp -l chinese_U17.bin translated_U17.bin   # или python-скрипт diff
```
Изменённые регионы = строки/глифы. К ним применяйте `rlad_decoder`. Это самый
надёжный путь для платформы A5 (вся приборка в 128 МБ, вся разница — в этих чипах).
