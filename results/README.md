# results — примеры успешных находок

Витрина того, что **реально извлечено из сырого NOR-дампа** приборки Geely Coolray
(SX11-A5). Это не рендеры-заготовки, а настоящая графика приборки, прочитанная напрямую
из дампов.

Полное описание находки, метод и параметры — в
[../docs/SUCCESSFUL_FINDINGS.md](../docs/SUCCESSFUL_FINDINGS.md).

Файлы в этой папке — **вручную отобранная витрина** (чистая зона иконок `0x28F8000..0x2940000`).
Сам инструмент пишет рендеры в `output/found_glyphs/` (временная папка, в git не входит);
отсюда лучшие панорамы скопированы сюда. Воспроизвести рендер:
`python tools/extract_raw_a8.py Coolray_25_Dash_U17.bin --zone 0x28F8000 0x2940000 --width 48`.

## Файлы

| Файл | Что это |
|---|---|
| `U17_icons_0x28F8000_w48.png` | **Галерея иконок с чипа U17** (ноты, авто, спидометры, замки, предупреждения и т.д.) |
| `U18_icons_0x28F8000_w48.png` | Галерея с чипа U18 — тот же набор (подтверждает воспроизводимость) |

## Ключевой факт

Иконки телльтейлов приборки хранятся в NOR **несжатым сырым A8** (1 байт = 1 пиксель alpha),
а НЕ в проприетарном RLAD. Поэтому их удалось прочитать напрямую — без таблицы разметки из
закрытого MCU и без RLAD-декодера.

- **Зона чистых иконок:** ~`0x28F8000 .. 0x2940000` (не жёсткий интервал — иногда удобнее
  стартовать с `0x28F4000`; пробуй оба). Границы между U17/U18 чуть различаются.
- **Формат:** raw A8, ширина строки (stride) = **48 пикселей** (базовая ячейка сетки = 24px)
- Иконки **продублированы на обоих чипах** (U17 и U18) — набор один и тот же

Всё остальное (текст, цвет, крупная графика) — **RLAD-сжато** и вслепую не извлекается
(нужен ключ: таблица разметки из MCU через JTAG XIP, либо diff двух языковых дампов).
